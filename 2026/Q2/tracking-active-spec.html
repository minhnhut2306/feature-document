User Story
Tracking Active — Ghi nhận trạng thái hoạt động khi mở app
As a admin hệ thống Arena,
I want to tự động ghi nhận activity mỗi khi nhân viên/chủ club mở app Arena Scoreboard,
So that tôi có thể theo dõi khách hàng nào đang thực sự sử dụng app tại club.

Acceptance Criteria

1 Ghi nhận activity thành công

GIVEN User mở app Arena Scoreboard và vào màn hình Home
WHEN App gọi API tracking với action = "Open App"
THEN BE nhận request, ghi activity vào database thành công

2 Ghi nhận mỗi lần mở app

GIVEN User đã mở app trước đó và đóng app
WHEN User mở lại app lần nữa
THEN Hệ thống ghi nhận một bản ghi activity mới

3 Xử lý khi API lỗi

GIVEN User mở app nhưng không có kết nối mạng hoặc server lỗi
WHEN App gọi API tracking thất bại
THEN App vẫn hoạt động bình thường, không ảnh hưởng trải nghiệm user
AND Không ghi activity vào database

4 Admin xem được activity

GIVEN Hệ thống đã ghi nhận activity từ nhiều thiết bị/club
WHEN Admin truy cập dashboard hoặc báo cáo
THEN Admin thấy được danh sách các thiết bị/club có hoạt động
Flow chính — Tracking Active khi mở app
USER1. Mở app Arena Scoreboard
User (nhân viên/chủ club) mở app trên thiết bị tại club. App khởi động và load màn hình Home.
Home loaded
SYSTEM2. Gọi API Tracking
App tự động gọi API đến BE với action = "Open App", kèm thông tin thiết bị (device ID, club ID, timestamp).
API Request
API response?
SUCCESS3a. Ghi activity thành công
BE nhận request → validate → ghi bản ghi activity vào database với thông tin: device ID, club ID, action, timestamp. Trả về 200 OK.
FAILURE3b. API lỗi / Mất kết nối
Server không phản hồi hoặc trả lỗi. App không bị ảnh hưởng — user vẫn sử dụng bình thường. Activity không được ghi nhận.
USER4. Tiếp tục sử dụng app
User tiếp tục sử dụng app bình thường tại màn hình Home. Tracking xảy ra ngầm, không có popup hay thông báo nào hiển thị cho user.
Async / Background
SYSTEM5. Admin xem báo cáo activity
Admin truy cập hệ thống quản lý, xem danh sách các thiết bị/club đã có activity → biết được khách hàng nào đang sử dụng app.

TC-01 HAPPY P1 Mở app lần đầu — ghi activity thành công 1. Mở app Arena Scoreboard
2. Chờ Home screen load xong
3. Kiểm tra database		 Database có bản ghi activity với action = "Open App", device ID, club ID, timestamp chính xác
TC-02 HAPPY P1 Mở app nhiều lần — mỗi lần ghi 1 bản ghi 1. Mở app → đóng app
2. Mở lại app
3. Kiểm tra database		 Database có 2 bản ghi activity riêng biệt với timestamp khác nhau
TC-03 HAPPY P1 API trả về 200 OK 1. Mở app với kết nối mạng ổn định
2. Monitor API response		 API trả về status 200, response body hợp lệ
TC-04 HAPPY P2 Activity chứa đầy đủ thông tin 1. Mở app
2. Query database lấy bản ghi mới nhất		 Bản ghi chứa: device_id, club_id, action ("Open App"), created_at (timestamp)
TC-05 HAPPY P2 Admin xem được activity trên dashboard 1. Mở app từ thiết bị
2. Đăng nhập admin dashboard
3. Xem activity log		 Admin thấy bản ghi activity tương ứng với thiết bị/club vừa mở app
TC-06 NEGATIVE P1 Mở app khi không có mạng 1. Tắt WiFi / 4G
2. Mở app Arena Scoreboard
3. Quan sát app hoạt động		 App vẫn mở bình thường, không crash. Activity không được ghi vào DB
TC-07 NEGATIVE P1 Server BE bị down 1. Stop BE server
2. Mở app
3. Quan sát app hoạt động		 App không bị ảnh hưởng, user sử dụng bình thường. Không có error hiển thị
TC-08 NEGATIVE P2 API trả về lỗi 500 1. Mock API trả 500
2. Mở app
3. Quan sát UI		 App hoạt động bình thường, không hiển thị error cho user
TC-09 NEGATIVE P2 API timeout 1. Giả lập network delay > timeout threshold
2. Mở app
3. Quan sát		 App không bị block bởi API call. Home screen load bình thường
TC-10 NEGATIVE P2 Database connection lỗi 1. Ngắt kết nối DB
2. Mở app
3. Kiểm tra BE log		 BE log error, app user-side không bị ảnh hưởng
TC-11 PERF P2 API response time 1. Mở app
2. Đo thời gian API tracking response		 API response ≤ 500ms trong điều kiện mạng bình thường
TC-12 PERF P2 API không block UI 1. Mở app
2. Đo thời gian Home screen hiển thị
3. So sánh có/không có tracking		 Home screen load time không bị ảnh hưởng đáng kể (API gọi async/background)
TC-13 PERF P3 Nhiều thiết bị mở app đồng thời 1. 50+ thiết bị mở app cùng lúc
2. Kiểm tra DB records		 Tất cả activity đều được ghi nhận, không mất data
TC-14 EDGE P3 App từ background lên foreground 1. Mở app (đã tracking)
2. Nhấn Home, app vào background
3. Mở lại app từ background		 Xác định rõ: có ghi activity mới khi resume từ background hay không (theo business rule)
