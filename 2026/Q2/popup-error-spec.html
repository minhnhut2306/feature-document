User Story
Hiển thị thông báo lỗi dạng Pop-up trong App Arena Bảng điểm
Với vai trò là nhân viên vận hành / chủ club sử dụng app Arena bảng điểm,
Tôi muốn khi app gặp lỗi (mất mạng, bluetooth ngắt, camera lỗi, ...) sẽ hiển thị một popup thông báo rõ ràng thay vì chỉ hiện toast nhỏ ở góc màn hình,
Để tôi biết chính xác lỗi gì đang xảy ra và có thể xử lý ngay mà không bỏ lỡ thông báo.

Acceptance Criteria

1Popup Bluetooth & API

GIVENApp đang hoạt động bình thường
WHENBluetooth ngắt kết nối hoặc API request thất bại
THENHiện popup màu vàng với nút "Thử lại"

2Popup Mất mạng

GIVENThiết bị mất kết nối Internet
WHENHệ thống phát hiện mất mạng
THENHiện popup đỏ, cảnh báo khóa sau 3 ngày, nút "Kết nối lại"

3Popup Livestream & Camera

GIVENTính năng livestream hoặc camera đang hoạt động
WHENLivestream thất bại hoặc camera mất kết nối
THENHiện popup đỏ, hướng dẫn kiểm tra, nút "OK"

4Popup Chưa cài đặt

GIVENUser truy cập tính năng chưa được cấu hình
WHENTính năng yêu cầu cài đặt trước
THENHiện popup xanh, hướng dẫn vào Settings để cấu hình
Luồng xử lý Pop-up lỗi
USER1. Đang sử dụng Bảng điểm
Nhân viên đang thao tác trên app Arena — chấm điểm, xem camera, livestream, v.v.
Lỗi xảy ra
SYSTEM2. Phát hiện lỗi
App tự động detect lỗi: Bluetooth ngắt, Mất mạng, API fail, Livestream fail, Camera mất, hoặc Chưa cài đặt.
⬥ Phân loại lỗi?
SYSTEM3. Hiển thị Popup tương ứng
Popup hiện đè lên giao diện bảng điểm với overlay mờ phía sau.
• Vàng: Bluetooth ngắt, API lỗi → nút Thử lại
• Đỏ: Mất mạng (cảnh báo khóa 3 ngày), Livestream fail, Camera mất → nút Kết nối lại / OK
• Xanh: Chưa cài đặt → nút Đi Settings
USER4. Bấm nút hành động trên Popup
User đọc thông báo lỗi, bấm nút tương ứng: Thử lại, Kết nối lại, OK, hoặc Đi Settings.
⬥ Kết quả xử lý?
✓ OK5a. Xử lý thành công
Kết nối lại OK, đóng popup, quay về bảng điểm bình thường.
✗ FAIL5b. Vẫn lỗi
Kết nối thất bại — popup vẫn hiện hoặc hiện lại để user thử lần nữa.
⚠️ Nếu mất mạng quá 3 ngày, hệ thống sẽ tự khóa thiết bị.

TC-01 HAPPY P1 Popup Bluetooth ngắt hiện đúng 1. Mở app Arena, bắt đầu game
2. Tắt Bluetooth trên thiết bị		 Popup màu vàng hiện với nội dung "Bluetooth ngắt kết nối" và nút "Thử lại"
TC-02 HAPPY P1 Popup mất mạng hiện đúng 1. Mở app Arena
2. Tắt WiFi và mobile data		 Popup đỏ hiện, cảnh báo khóa thiết bị sau 3 ngày, nút "Kết nối lại"
TC-03 HAPPY P1 Popup API lỗi hiện đúng 1. App đang hoạt động
2. API server trả về lỗi (500/timeout)		 Popup vàng hiện với nút "Thử lại", bấm "Thử lại" → gọi lại request
TC-04 HAPPY P2 Popup chưa cài đặt hiện đúng 1. Truy cập tính năng chưa cấu hình
2. (VD: bật livestream khi chưa set RTMP)		 Popup xanh hiện, hướng dẫn vào Settings
TC-05 HAPPY P1 Đóng popup sau khi xử lý lỗi 1. Trigger lỗi → popup hiện
2. Khắc phục lỗi (bật lại WiFi, Bluetooth...)
3. Bấm nút hành động trên popup		 Popup đóng, quay về bảng điểm bình thường
TC-06 NEGATIVE P1 Lỗi xảy ra nhưng không hiện popup 1. Trigger từng loại lỗi
2. Quan sát UI		 KHÔNG ĐƯỢC XẢY RA — tất cả lỗi phải hiện popup tương ứng
TC-07 NEGATIVE P2 Popup hiện sai màu sắc 1. Trigger lỗi Bluetooth → kiểm tra popup vàng
2. Trigger mất mạng → kiểm tra popup đỏ
3. Trigger chưa cài đặt → kiểm tra popup xanh		 Mỗi loại lỗi hiện đúng màu popup đã quy định
TC-08 NEGATIVE P2 Nút "Thử lại" không hoạt động 1. Trigger lỗi API → popup vàng hiện
2. Bấm "Thử lại"		 Request phải được gọi lại. Nếu thành công → đóng popup. Nếu thất bại → popup hiện lại.
TC-09 NEGATIVE P1 Nhiều lỗi xảy ra cùng lúc 1. Tắt WiFi + tắt Bluetooth cùng lúc
2. Quan sát popup		 Hiện popup lỗi ưu tiên cao nhất trước (mất mạng), sau khi xử lý xong → hiện popup tiếp theo nếu còn
TC-10 NEGATIVE P2 Popup mất mạng — đếm ngược 3 ngày 1. Tắt mạng liên tục 3 ngày
2. Kiểm tra trạng thái thiết bị		 Sau 3 ngày mất mạng liên tục, thiết bị tự khóa đúng như cảnh báo
TC-11 HARDWARE P1 Camera mất kết nối giữa game 1. Đang chơi game, camera đang stream
2. Rút dây nguồn camera		 Popup đỏ hiện với hướng dẫn kiểm tra nguồn, dây kết nối, IP camera
TC-12 HARDWARE P2 Bluetooth bật lại → tự kết nối 1. Bluetooth ngắt → popup vàng hiện
2. Bật Bluetooth lại
3. Bấm "Thử lại"		 Popup đóng, kết nối Bluetooth khôi phục thành công
TC-13 HARDWARE P2 Livestream fail khi mạng yếu 1. Kết nối WiFi tốc độ rất thấp
2. Bắt đầu livestream		 Popup đỏ "Livestream thất bại" hiện, nút OK để đóng
TC-14 PERFORMANCE P2 Popup hiện nhanh sau khi lỗi xảy ra 1. Trigger lỗi (tắt WiFi, rút camera...)
2. Đo thời gian popup hiện		 Popup phải hiện trong vòng 2-3 giây sau khi lỗi được phát hiện
TC-15 PERFORMANCE P3 Popup không ảnh hưởng đến game đang chơi 1. Đang chơi game (có điểm)
2. Trigger lỗi API → popup hiện
3. Bấm "Thử lại" → popup đóng		 Điểm số, timer, trạng thái game không bị thay đổi sau khi popup đóng
